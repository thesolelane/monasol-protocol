# MonaSol Protocol Security System — Complete Package

## 📦 Package Contents

This package contains the complete design documentation for the MonaSol Protocol Security System, connecting your admin dashboard to on-chain security layers while preserving MonaSol's core principle: **the protocol never holds keys, never holds NFTs, and has no custody of unlock credentials.**

## 🔐 Core Security Principles

| Principle | Implementation |
|-----------|----------------|
| **No Admin Keys** | Dashboard is read-only; all actions require wallet signature |
| **Multi-Sig Required** | 3-of-5 threshold for any state change + 24-72h timelock |
| **Economic Security** | Watchers stake 10,000 MSL; slashed for false positives |
| **Transparency** | All events emitted on-chain, indexed by Subgraph |
| **User Sovereignty** | Self-mode vaults cannot be locked by protocol |
| **Algorithmic Rotation** | Node rotation is mandatory and automatic, not discretionary |
| **Collision Guard** | Failover and rotation cannot occur simultaneously |

## ⚠️ Known Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Solana light client lag | Medium | High | Health oracle auto-detects; pause if >12 slots |
| Watcher centralization | Medium | High | Staking caps; random selection for critical alerts |
| Multi-sig compromise | Low | Critical | Geographic distribution; hardware wallets; 3-of-5 threshold |
| Dashboard server breach | Medium | Low | No keys stored; all actions require wallet signature |
| Notification spam | Medium | Medium | Rate limiting; user filters; deduplication |
| Regulatory scrutiny | Medium | High | Legal review before MSL distribution; non-US launch first |
